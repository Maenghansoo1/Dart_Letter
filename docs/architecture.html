02 — ARCHITECTURE

시스템 아키텍처

Flutter + FastAPI + Supabase 기반의 기업분석 앱 전체 구성도

FRONTEND
Flutter
iOS / Android 동시 개발. Riverpod 상태관리, go_router 라우팅
BACKEND
FastAPI
Python 비동기 API 서버. DART, 네이버, KRX API 통합 처리
DATABASE
Supabase
PostgreSQL + 인증 + Realtime. RLS로 보안 처리
AI
Gemini API
공시 원문 쉬운 설명 + 3줄 요약 생성. 무료 플랜 활용
DATA API
DART / 네이버 / KRX
공시 원문, 뉴스, 주가 데이터. 전부 무료 API 활용
NOTIFICATION
Firebase FCM
공시 알림, 뉴스 알림, 커뮤니티 댓글 알림 푸시

핵심 데이터 흐름
01
사용자가 종목 검색
Flutter 탐색 화면에서 종목명 또는 카테고리로 검색. FastAPI가 Supabase에서 캐시 확인 후 없으면 KRX API 호출.
Flutter → FastAPI → Supabase / KRX
02
종목 상세 페이지 진입
DART API에서 최신 공시 목록 조회. 동시에 네이버 API에서 기업명으로 관련 뉴스 검색. 결과는 Supabase에 1시간 캐싱.
DART API + 네이버 API → Supabase 캐시
03
AI 공시 분석 요청
공시 원문을 Gemini API로 전송. "쉬운 설명 3~5문장 + 핵심 요약 3줄" 형식으로 반환. 결과는 영구 캐싱.
Gemini API → 영구 캐시 저장
04
관심종목 등록 및 알림
즐겨찾기 등록 시 Supabase watchlist 저장. 새 공시 감지 시 FCM으로 푸시 알림 전송.
Supabase → FCM → Flutter 알림
05
커뮤니티 실시간 댓글
Supabase Realtime으로 새 댓글 즉시 반영. 별도 polling 없이 WebSocket 기반으로 실시간 업데이트.
Supabase Realtime WebSocket
